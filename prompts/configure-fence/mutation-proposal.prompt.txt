Produce the exact Fence configuration mutation for the selected scope.

Call tool %%TOOL_NAME%% exactly once with JSON arguments only.
Do not output prose.

Required output fields:
- requestValidity: "valid" or "invalid"
- reasoning: non-empty string
- changeMode: non-empty string
- effectSummary: non-empty string
- conflictSummary: optional string (use "none" if no conflict)
- invalidReason: required when requestValidity is "invalid"
- mutationType: required when requestValidity is "valid"
- writeContent: required when mutationType is "write"
- edits: required when mutationType is "edit"

Mutation semantics:
- Invalid mode:
  - if the request is vague, nonsense, or non-actionable, set
    requestValidity="invalid".
  - include concise invalidReason.
  - do not invent or propose config changes.
- Write mode:
  - writeContent must be the full final file content.
  - use for create/full rewrite or when edit safety is uncertain.
- Edit mode:
  - each edit must contain oldText and newText.
  - each oldText must uniquely match exactly one region of existing
    content.
  - edits must be non-overlapping.
  - use only for safe minimal patching.

Fence config requirements:
- Final content must be valid JSON object.
- No comments, no trailing commas.
- Fence extends is allowed and may use any Fence-supported value.
- Preserve unrelated settings unless user explicitly requests removal.

Selection rules:
- If request is not actionable, use requestValidity="invalid".
- If target file does not exist, use mutationType="write".
- If an edit cannot be guaranteed unique/non-overlapping, use
  mutationType="write".
